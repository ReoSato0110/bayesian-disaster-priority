# 災害支援優先度・参考指標アプリ（デモ）

## ⚠️ 重要な注意事項
本アプリは **デモ・学習目的**で作成されたものです。

- 実際の災害対応や意思決定を **自動化するものではありません**
- 表示されるデータは **すべて架空のサンプルデータ**です
- 実運用・実災害への利用は **想定していません**

実際の支援判断は、専門家の知見や現場状況を総合的に考慮して行う必要があります。

---

## アプリ概要

本アプリは、  
**「過去データの不確実性」と「緊急度」を分離して扱うことで、  
災害支援要請の優先度を“参考指標”として可視化する**  
Streamlit デモアプリです。

数値はあくまで「考えるための材料」であり、  
判断を代替するものではありません。

---

## なぜこのアプリを作ったか

災害時の情報には、以下のような課題があります。

- 過去の実績が少ない要請は、単純な割合では評価しにくい
- 緊急度の高低と、データの信頼性が混同されやすい
- 数値が「判断の根拠」ではなく「判断そのもの」になりがち

本アプリでは、

- **過去データの不確実性**
- **現在の緊急度**

を分けて扱い、  
「意思決定を補助する参考指標」としてスコアを提示します。

---

## このアプリが「しないこと」

- 自動的に支援の可否を決定しません
- 行政・現場オペレーションを代替しません
- 実データの収集・解析は行いません

あくまで  
**「考えるための数値を提示する」**  
デモアプリです。

---

## 優先度スコアの考え方（概要）

各支援要請には、以下の情報が設定されています。

- 過去に支援が必要だった回数
- 観測・報告された総回数
- 現在の緊急度（0〜1）

過去データについては、
単純な割合ではなく **ベイズ的推定**を用いることで、

- データが少ない場合の過信を防ぎ
- 不確実性を考慮した期待値

を計算しています。

その結果に、  
現在の **緊急度を重みとして掛け合わせ**、  
参考用の優先度スコア（0〜100）として表示します。

> ⚠️ このスコアは「正解」ではありません  
> 人間が考えるための補助指標です
